# `match`: сопоставление с образцом

> Ступень 2 · Урок B. Парный файл: [`enums.md`](./enums.md) — про значения, которые мы здесь разбираем.

`match` — главный инструмент разбора enum'ов (включая `Option` и `Result`). Это место спотыкания для C/Python-мозга, поэтому разбираем медленно.

---

## 1. Рабочий пример

```rust
fn area(shape: &Shape) -> f64 {
    match shape {
        Shape::Circle { radius } => std::f64::consts::PI * radius * radius,
        Shape::Rectangle { width, height } => width * height,
        Shape::Triangle(a, b, c) => {
            let s = (a + b + c) / 2.0;
            (s * (s - a) * (s - b) * (s - c)).sqrt()   // формула Герона
        }
        Shape::Point => 0.0,
    }
}
```

`Shape` — это enum из [`enums.md`](./enums.md). Дальше разбираем, что здесь на самом деле происходит.

---

## 2. Главное недопонимание: «`shape` нигде не используется»

Соблазн — прочитать `match shape` как C'шный `switch(x)`, где слева от `case` стоит **значение**, с которым `x` сравнивают по `==`. В Rust это **не так**.

Слева от стрелки `=>` стоит **паттерн** (pattern) — другая грамматическая категория. Паттерн не сравнивается с `shape` по равенству. Паттерн — это **шаблон формы**, и вопрос не «равно ли», а «**подходит ли `shape` под эту форму, и если да — разбери его на части**».

Поэтому `shape` используется в самом главном месте: он — то, **против чего по очереди примеряется каждый паттерн**. Просто это «использование» выглядит не как `shape.something`, а как сопоставление со структурным шаблоном слева.

Тот же `match` псевдокодом:

```text
для значения, лежащего в shape:
    это Circle?       → да: достань поле radius в переменную radius; верни PI*radius*radius
    иначе Rectangle?  → да: достань width и height;                  верни width*height
    иначе Triangle?   → да: достань три числа в a, b, c;             верни (формула Герона)
    иначе Point?      → да:                                          верни 0.0
```

Запомни различие одной строкой: **в `switch` слева стоят значения, которые сверяют. В `match` слева стоят паттерны, которые примеряют — и они же распаковывают.**

## 3. Второй сдвиг: `radius` — это выход, а не вход

Легко подумать, что `radius` в `Shape::Circle { radius }` — существующая переменная, которую с чем-то сверяют. **Нет.** `radius` **создаётся прямо здесь** — это новое имя, в которое паттерн **кладёт** значение, вытащенное из `shape`.

Стрелка работает в два хода одним движением:

```rust
Shape::Circle { radius } => std::f64::consts::PI * radius * radius
// ход 1: проверка   — "shape сейчас это Circle?"
// ход 2: привязка   — если да, поле circle.radius извлекается в новую переменную radius
// правая часть      — radius уже доступен как обычная локальная переменная
```

Это называется **деструктуризация**, и она тебе знакома. В Python ты пишешь то же самое:

```python
a, b = (1, 2)        # a и b СОЗДаются и наполняются из правой части, а не сверяются
```

```python
match shape:
    case Circle(radius):     # radius тут тоже создаётся
        return pi * radius ** 2
```

`match`-ветка — это тот же анпакинг, только со встроенной проверкой «а точно ли значение имеет нужную форму». Деструктуризация + условие, слитые в одно.

Поток данных целиком:

```text
shape = Circle { radius: 5.0 }
   │
   ▼  match примеряет паттерн Shape::Circle { radius } — форма совпала ✔
   │
   ▼  поле 5.0 привязывается к новой переменной radius
   │
   ▼  правая часть: PI * radius * radius  →  PI * 5.0 * 5.0
```

Данные `shape` именно что используются — они вытекают из него через паттерн в переменные, а из переменных в вычисление. Написать `shape.radius` напрямую ты бы и не смог: пока компилятор не доказал через `match`, что `shape` сейчас точно `Circle`, лезть в это поле запрещено. Это и есть та безопасность — тег и доступ к данным склеены.

## 4. Три вещи, которых нет у C++ `switch`

Три вещи, которых нет у `switch`:

1. **Деструктуризация в ветке** — разобрано выше: паттерн одновременно проверяет вариант и достаёт его данные в локальные переменные.
2. **Проверка на исчерпывающность.** Если дописать в `enum Shape` вариант `Pentagon`, а в `match` его не обработать — код **не скомпилируется**. Не warning, как в C++, а ошибка. Добавляешь состояние в систему — компилятор гонит тебя по всем местам, где это состояние надо учесть. Рефакторинг перестаёт быть рулеткой. В C++ забытый `case` тихо проваливается в `default` и багует в проде через полгода.
3. **Богатый язык паттернов.** Несколько приёмов, которые будешь использовать постоянно:

```rust
let n = 7;
match n {
    0 => println!("ноль"),
    1..=9 => println!("одна цифра"),                 // диапазон
    x if x % 2 == 0 => println!("чётное {}", x),     // guard — условие на ветку
    _ => println!("прочее"),                          // _ ловит остаток (как default)
}
```

`_` — это «всё остальное». **Но не злоупотребляй им на своих enum'ах.** `_` отключает проверку исчерпывающести: добавишь новый вариант — компилятор промолчит, потому что `_` его поймал. На внешних и числовых типах `_` нужен; на своих доменных enum'ах перечисляй варианты явно — это твоя страховка при рефакторинге.

## 5. Короткие формы: `if let` и `let ... else`

Когда полный `match` избыточен (часто так и есть при разборе `Option`/`Result`), есть две лаконичные формы:

```rust
// if let — "разбери только этот один вариант"
if let Some(c) = first_char(input) {
    println!("{}", c);
}   // остальные варианты молча игнорируются
```

```rust
// let ... else — разбери счастливый путь, а несчастливый обязан "убежать"
let Some(c) = first_char(input) else {
    println!("пусто, выходим");
    return;          // обязателен выход: return / break / panic!
};
// сюда поток дойдёт только если был Some — c уже распакован и доступен ниже
println!("{}", c);
```

`let ... else` спасает от лестницы вложенности: распаковал нужное — иначе вышел, дальше код пишется «плоско». (`first_char` и `Some` — из [`enums.md`](./enums.md).)

## 6. Тонкость про ссылки (на будущее, без паники)

В примере `area` значение `shape` имеет тип `&Shape` — мы матчим по ссылке. Поэтому `radius`, `width` и прочие привязываются тоже как ссылки (`&f64`), а не как копии. Компилятор делает это автоматически (фича называется «match ergonomics»), и арифметика с ними работает прозрачно.

Запомни только смысл: матч по `&Shape` не «вынимает с потрохами» оригинал, а даёт ссылки внутрь него — `shape` остаётся цел. Логично: мы передали его по ссылке именно чтобы не забирать владение. Детали владения — на Ступени 3.

## Мини-упражнение на `match`

Без нового проекта, можно в `fn main` песочницы:

1. Заведи `let xs = [0, 3, 7, 12, -4];`. Пройди циклом и для каждого числа через `match` напечатай: `"ноль"`, `"одна цифра"` (диапазон `1..=9`), `"отрицательное"` (guard `x if x < 0`), иначе `"большое"`.
2. Возьми `Option<i32>` со значением `Some(10)` и распакуй его двумя способами: через полный `match` и через `if let`. Убедись, что результат одинаков.
3. Перепиши вариант с `if let` на `let ... else`, чтобы при `None` функция делала ранний `return`.

## Шпаргалка

- **match** примеряет *паттерны* (не значения): проверяет форму И распаковывает данные за один ход.
- Имена в паттерне (`radius`, `a`) — **создаются** привязкой, это выход, а не вход.
- **Исчерпывающесть** проверяется компилятором: забыл вариант — ошибка сборки.
- Паттерны умеют: диапазоны `1..=9`, guard'ы `x if ...`, остаток `_`.
- `_` на своих enum'ах — осторожно: глушит проверку исчерпывающести.
- **if let** — разобрать один вариант; **let ... else** — разобрать счастливый путь, иначе «убежать».

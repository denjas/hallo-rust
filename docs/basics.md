# Базовый синтаксис Rust

> Ступень 1. Для тех, кто знает C/C++/Python — здесь только **дельта**, а не «что такое переменная». Парные файлы ступени 2: [`enums.md`](./enums.md), [`match.md`](./match.md).

Сразу снимем путаницу: в Rust `{}` означает **две разные вещи**.

1. **Блок кода** — `{ ... }`. Про него весь раздел 4, и это главное в этом уроке.
2. **Плейсхолдер форматирования** — внутри строк: `println!("{}", x)`. Это не блок, это «подставь сюда значение». Не путай с первым.

---

## 1. Переменные: `let`, `mut`, `const`, shadowing

```rust
let x = 5;          // неизменяемая ПО УМОЛЧАНИЮ. Это не const — см. ниже
// x = 6;           // ОШИБКА компиляции
let mut y = 5;      // вот теперь изменяемая
y = 6;              // ок

let z: i64 = 10;    // явный тип; без аннотации тип выводится (как auto в C++)
```

**`let` без `mut` — это не `const`.** Изменяемость в Rust *opt-in*: по умолчанию переменную нельзя менять, и `mut` надо дописать осознанно. Это переворот по сравнению с C++, где надо не забыть `const`. Забыть `mut` безопасно — компилятор напомнит.

`const` — это отдельная штука: **именованное значение, зафиксированное на этапе компиляции**. Компилятор вычисляет его при сборке и **подставляет (инлайнит) прямо в каждое место использования**, будто ты везде вписал это число руками.

```rust
const MAX_USERS: u32 = 10_000;       // подчёркивания в числах — разделитель разрядов, для читаемости
const BUFFER_SIZE: usize = 64 * 1024; // выражение можно, но оно обязано считаться при компиляции
```

Три обязательных свойства:

1. **Всегда с типом** — вывод типа здесь не работает, в отличие от `let`.
2. **Значение известно на этапе компиляции** — никакого ввода с клавиатуры или чтения файла, только литералы, арифметика над ними и `const fn`.
3. **Имя в SCREAMING_SNAKE_CASE** — такова конвенция.

В отличие от `let`, `const` можно объявить и на верхнем уровне модуля (глобально), а не только внутри функции. Нужен он, чтобы дать имя «магическим числам» и зафиксировать настройки — лимиты, размеры буферов, постоянные: `if users > MAX_USERS` читается понятнее, чем `if users > 10000`, и значение правится в одном месте.

По смыслу это **как `#define` в C, но безопасный и с типом**. Разница принципиальная: `#define` — это текстовая замена препроцессором, не знающая ни про типы, ни про синтаксис, отсюда классические грабли вроде `#define SQUARE(x) x*x`, где `SQUARE(1+2)` разворачивается в `1+2*1+2 == 5`. `const` подставляет не сырой текст, а уже типизированное значение, разобранное компилятором, — поэтому такого класса багов с ним нет, тип проверяется, и работают обычные правила областей видимости. То есть `const` делает то, *зачем* в C тянутся к `#define`, но не *так, как* `#define`. (Аналогия с `constexpr` из C++ тоже точная: и там, и там смысл один — «значение считает компилятор, а не рантайм».)

**Shadowing** — этого нет в C++/Python: можно «переобъявить» переменную тем же именем, в т.ч. с новым типом. Это не мутация, а новая переменная, которая закрывает старую:

```rust
let s = "42";                 // s: &str
let s: i32 = s.parse().unwrap();   // s: i32 — то же имя, новый тип. Идиоматично.
let s = s * 2;                // и снова новое s
```

## 2. Типы и отсутствие неявных преобразований

Базовые типы:

1. **Целые:** `i8 i16 i32 i64 i128` и беззнаковые `u8 ... u128`. Плюс `isize`/`usize` — размер указателя платформы; `usize` — тип для индексов и длин (аналог `size_t`). Литерал по умолчанию выводится в `i32`.
2. **Дробные:** `f32`, `f64` (по умолчанию `f64`).
3. **`bool`** — строго `true`/`false`.
4. **`char`** — **4 байта**, один Unicode-скаляр, а не байт как в C.

Ключевое отличие от C: **неявных числовых преобразований нет вообще.**

```rust
let a: i32 = 5;
let b: i64 = a;          // ОШИБКА: типы не совпадают
let b: i64 = a as i64;   // явное приведение через `as`
let b: i64 = a.into();   // или через типаж From/Into (про это на Ступени 4)
```

После молчаливых promotion'ов C это поначалу раздражает, потом понимаешь, сколько багов с переполнением и усечением это убивает.

## 3. Функции

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b        // возвращаемое значение — БЕЗ return и БЕЗ точки с запятой (см. раздел 4)
}

fn greet(name: &str) {   // нет `-> Тип` ⇒ функция возвращает () — «ничего»
    println!("Привет, {}", name);
}
```

Типы параметров и возврата **обязательны** — никакого вывода типов в сигнатуре (в отличие от тел функций, где он есть). Это сознательно: сигнатура — это контракт, и он должен быть явным.

## 4. 🎯 Блоки `{}` — это выражения

Вот то, ради чего урок. В C, C++ и Python блок `{ }` (или отступ в Python) — это просто **группа инструкций**. Он ничего не «возвращает», он только исполняет. В Rust всё иначе:

> **Почти всё в Rust — выражение. Блок `{ ... }` — это тоже выражение, и оно вычисляется в значение.**

### Главное правило

Блок вычисляется в значение своего **последнего выражения, если у того НЕТ точки с запятой**. Это последнее выражение называют *хвостовым* (tail expression).

```rust
let x = {
    let a = 2;
    let b = 3;
    a + b        // нет `;` ⇒ это хвостовое выражение ⇒ блок вычисляется в 5
};               // x == 5
```

Блок здесь делает две вещи сразу: создаёт область видимости (как в C — `a` и `b` снаружи не видны) **и** выдаёт наружу значение. Второго в C/C++/Python просто нет.

### Точка с запятой решает всё

Это центральная механика. `;` превращает выражение в **инструкцию** (statement) и **выбрасывает его значение**.

```rust
let x = {
    let a = 2;
    a + 3        // хвостовое выражение → x == 5
};

let y = {
    let a = 2;
    a + 3;       // ← точка с запятой! значение выброшено, хвостового выражения нет
};               // y == ()  ← блок вычислился в «ничего»
```

Разница между `x` и `y` — один символ `;`. У `x` блок вернул число, у `y` — `()` (про этот тип ниже). Запомни намертво: **`;` в конце строки = «выбросить значение этой строки».**

### Тип-пустышка `()` (unit)

`()` — это «юнит», тип-пустышка с единственным значением `()`. Это Rust'овый способ сказать «здесь нет осмысленного значения» — примерно как `void` в C, только это настоящий тип, который можно хранить и возвращать. Блок, заканчивающийся на `;`, вычисляется в `()`. Функция без `-> Тип` возвращает `()`.

### `if`/`else` — это выражение, а не оператор

Поэтому тернарного оператора `?:` в Rust нет — он не нужен:

```rust
let max = if a > b { a } else { b };   // обе ветки — блоки, каждая даёт значение
```

Обе ветки обязаны давать **один тип** (иначе чему равен `max`?). Отсюда следствие: `if` без `else`, использованный как значение, почти всегда ошибка — ведь «несуществующая» ветка `else` дала бы `()`:

```rust
let x = if cond { 5 };   // ОШИБКА: ветка-then даёт i32, отсутствующий else даёт ()
let x = if cond { 5 } else { 0 };   // ок
```

Когда `if` используется ради побочного эффекта (а не как значение), `else` не нужен — тогда весь `if` имеет тип `()`:

```rust
if cond {
    println!("да");      // тип всего if — (), значение никому не нужно
}
```

### `loop` — выражение, возвращающее значение через `break`

`loop` — бесконечный цикл, но из него можно «выкинуть» значение через `break`:

```rust
let result = loop {
    let n = next_attempt();
    if n.is_valid() {
        break n;          // break СО значением → это и станет значением всего loop
    }
};                        // result == n
```

Это уникально для `loop`. `while` и `for` всегда вычисляются в `()` — у них `break` значение нести не умеет.

### `match` — тоже выражение

Каждая ветка даёт значение, и весь `match` вычисляется в него (подробно в [`match.md`](./match.md)):

```rust
let label = match n {
    0 => "ноль",
    1..=9 => "цифра",
    _ => "много",
};   // label получает строку из сработавшей ветки
```

### Тело функции — это блок, а значит работает то же правило

Возврат из функции — это просто хвостовое выражение её блока. Поэтому `return` в конце не нужен:

```rust
fn square(n: i32) -> i32 {
    n * n        // хвостовое выражение = возвращаемое значение. БЕЗ `;`, БЕЗ `return`
}
```

`return` существует и нужен — но только для **раннего** выхода из середины функции:

```rust
fn checked_div(a: i32, b: i32) -> Option<i32> {
    if b == 0 {
        return None;     // ранний выход — тут return обязателен
    }
    Some(a / b)          // обычный путь — хвостовое выражение, без return
}
```

Стиль: ранний выход — через `return`, «нормальное» возвращаемое значение — хвостовым выражением. Писать `return n * n;` в конце не ошибка, но идиоматично — без `return`.

### ⚠️ Типичные ошибки новичка из C++/Python

Ошибка №1 — лишняя `;` на строке возврата:

```rust
fn square(n: i32) -> i32 {
    n * n;      // ← `;` выбросил значение; блок вернул (), а функция ждёт i32
}
// error[E0308]: mismatched types — expected `i32`, found `()`
```

Сообщение компилятора прямо говорит: «ожидал `i32`, получил `()`». Увидел такое — ищи лишнюю точку с запятой на последней строке.

Ошибка №2 — наоборот, забытая `;` в середине:

```rust
fn f() {
    let a = 1     // ← забыл `;`; компилятор думает, что выражение продолжается
    let b = 2;    // и спотыкается здесь
}
```

Правило в голове: **строки-действия в середине блока заканчиваются `;`; последняя строка-значение — без `;`.** Если функция ничего не возвращает (`-> ()`), то и последняя строка обычно с `;`.

## 5. Управление потоком

`if` — **без круглых скобок** вокруг условия, и условие **обязано** быть `bool`. Никакой «правдивости» чисел и указателей как в C/Python: `if x` при `x: i32` — ошибка, пиши `if x != 0`.

```rust
if n > 0 {
    ...
} else if n == 0 {
    ...
} else {
    ...
}
```

Циклы:

```rust
loop {              // бесконечный; выход через break (можно со значением)
    if done { break; }
}

while condition {   // как в C, но без скобок вокруг условия
    ...
}

for i in 0..5 {     // 0,1,2,3,4 — диапазон с исключённым концом (как range(5))
    println!("{}", i);
}
for i in 0..=5 {    // 0..5 включительно (..= — включающий диапазон)
    ...
}
for item in &collection {   // итерация по элементам — основной способ (про итераторы — Ступень 5)
    ...
}
```

`for` в Rust — это всегда «for-each» по итератору; сишного `for(i=0; i<n; i++)` с ручным счётчиком тут нет, диапазон `0..n` его заменяет.

**Метки циклов** — для `break`/`continue` из вложенного цикла наружу:

```rust
'outer: for x in 0..10 {
    for y in 0..10 {
        if x * y > 20 {
            break 'outer;    // выйти сразу из внешнего цикла
        }
    }
}
```

## 6. Мелочи, которые встретишь сразу

```rust
// однострочный комментарий
/* блочный */
/// док-комментарий (попадает в сгенерированную документацию)

println!("{}", x);          // {} — обычный вывод (трейт Display)
println!("{:?}", point);    // {:?} — отладочный вывод (трейт Debug, для своих структур нужен #[derive(Debug)])
println!("{x}");            // можно вставлять имя переменной прямо в скобки
println!("{} и {}", a, b);  // несколько плейсхолдеров по порядку

let text = String::from("владеющая строка");   // String — владеющая, в куче
let slice: &str = "строковый литерал";          // &str — заимствованный срез (про владение — Ступень 3)
```

`println!` с `!` — это **макрос**, а не функция (макросы умеют проверять формат на этапе компиляции). Пока просто пользуйся; про макросы — на Ступени 8.

## Шпаргалка

- `let` неизменяем по умолчанию; `mut` — opt-in; `const` — compile-time константа; **shadowing** переобъявляет имя (можно с новым типом).
- Неявных числовых преобразований **нет**: только `as` или `.into()`.
- Сигнатуры функций — с явными типами; тела — с выводом типов.
- **Блок `{ }` — это выражение**: его значение = последнее выражение **без** `;`.
- **`;` выбрасывает значение** строки; блок тогда вычисляется в `()` (unit, «ничего»).
- Возврат из функции = хвостовое выражение без `;`; `return` — только для раннего выхода.
- Лишняя `;` на строке возврата → ошибка `expected T, found ()`. Первое, что проверять.
- `if`/`match`/`loop` — выражения и дают значения; `loop` умеет `break value`, `while`/`for` — нет.
- `if`-условие — без скобок и строго `bool`; `for` — всегда for-each по итератору/диапазону.
- `{}` в строке `println!` — это плейсхолдер, не блок.

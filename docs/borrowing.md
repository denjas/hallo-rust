# Заимствование (borrowing)

> Ступень 3 · Урок B. Опирается на [`ownership.md`](./ownership.md). Парный файл: [`lifetimes.md`](./lifetimes.md).

В конце урока про владение мы упёрлись в неудобство: чтобы функция просто *посмотрела* на строку, ей приходилось отдавать владение и получать обратно. Решение — **заимствование**: даёшь временную **ссылку** на значение, не передавая владения. Владелец остаётся владельцем.

Ссылка — это `&`. Их два вида, и вся соль урока в правилах, по которым ими можно пользоваться.

---

## 1. Разделяемая ссылка `&T` — «дай посмотреть»

```rust
fn len_of(s: &String) -> usize {   // принимает ССЫЛКУ на String, а не саму String
    s.len()
}                                   // ссылка выходит из скоупа — но НИЧЕГО не уничтожается

fn main() {
    let text = String::from("привет");
    let n = len_of(&text);          // &text — одалживаем ссылку, владение остаётся у text
    println!("{text} = {n}");       // text по-прежнему наш, доступен
}
```

Сравни с версией из `ownership.md`, где `consume(text)` забирала владение и `text` становился недоступен. Здесь `&text` — это лишь заём: функция читает значение и возвращает, владелец не меняется.

Ключевое: **ссылка не владеет.** Когда ссылка выходит из области видимости, по ней ничего не уничтожается — она лишь указывала на чужое значение. `Drop` срабатывает у владельца, а не у того, кто одолжил. (Поэтому, кстати, `&T` — это `Copy`-тип из [`ownership.md`](./ownership.md): копировать указатель безопасно, double-free не грозит.)

По разделяемой ссылке нельзя **менять** значение — только читать:

```rust
fn main() {
    let text = String::from("привет");
    let r = &text;
    // r.push_str("!");   // ОШИБКА: через &T изменять нельзя
}
```

---

## 2. Изменяемая ссылка `&mut T` — «дай поменять»

Чтобы функция могла *изменить* значение, не забирая его, нужна изменяемая ссылка `&mut`:

```rust
fn append_bang(s: &mut String) {   // изменяемая ссылка
    s.push_str("!");               // меняем чужое значение по ссылке
}

fn main() {
    let mut text = String::from("привет");   // владелец обязан быть mut
    append_bang(&mut text);                   // одалживаем изменяемую ссылку
    println!("{text}");                       // "привет!"
}
```

Два требования: сам владелец объявлен `mut`, и ссылку берём явно как `&mut`. Логично — нельзя дать право на изменение того, что и сам владелец менять не может.

---

## 3. Правила заимствования — сердце borrow checker

Вот два правила, на которых держится вся безопасность Rust. Заучи:

1. **В каждый момент времени можно иметь ЛИБО любое число разделяемых ссылок `&T`, ЛИБО ровно одну изменяемую `&mut T` — но не то и другое сразу.**
2. **Любая ссылка обязана быть валидной** (указывать на живое значение; висячих ссылок не существует).

Правило 1 коротко: **«разделение ИЛИ изменение, но не одновременно»** (aliasing XOR mutability). Либо много читателей и никто не пишет, либо один писатель и больше никого.

```rust
let mut x = 5;

let a = &x;        // ок: разделяемая
let b = &x;        // ок: ещё одна разделяемая — много читателей можно
// let c = &mut x; // ОШИБКА: пока живут a и b, изменяемую брать нельзя

println!("{a} {b}");
```

```rust
let mut x = 5;
let m = &mut x;    // одна изменяемая
// let n = &mut x; // ОШИБКА: вторую изменяемую нельзя
// let r = &x;     // ОШИБКА: и разделяемую одновременно с изменяемой нельзя
*m += 1;           // *m — разыменование: добраться до самого значения
```

---

## 4. Зачем эти правила: что они вырезают на корню

Это не бюрократия — каждое правило убивает конкретный класс багов, который в C++ ловится в лучшем случае в рантайме, в худшем — никогда.

**Гонки данных (data races).** Гонка — это когда два пути одновременно лезут к одним данным и хотя бы один пишет. Правило «один писатель XOR много читателей» делает гонку **невыразимой в типах**. Отсюда «бесстрашная многопоточность» Rust (Ступень 7): компилятор не пустит код, который мог бы гонять данные.

**Инвалидация итераторов — классика C++.** Смотри:

```rust
let mut v = vec![1, 2, 3];
let first = &v[0];     // разделяемая ссылка ВНУТРЬ вектора
v.push(4);             // ОШИБКА: нельзя изменять v (push), пока жива ссылка first
println!("{first}");
```

```text
error[E0502]: cannot borrow `v` as mutable because it is also borrowed as immutable
```

В C++ `v.push_back(4)` может перевыделить буфер и сделать `first` висячим указателем — use-after-free, который рванёт «иногда». Rust ловит это **на этапе компиляции**: пока есть ссылка внутрь вектора, менять вектор запрещено. Тот самый инвариант, который C++-разработчик держит в голове, тут стережёт компилятор.

---

## 5. Заём кончается на последнем использовании (NLL)

Правила звучат строго, но на практике мягче, чем кажется: **заём живёт не до конца блока, а до последнего использования ссылки.** Это называется non-lexical lifetimes. Поэтому такой код легален:

```rust
let mut x = 5;

let a = &x;
println!("{a}");   // последнее использование a — заём a здесь и заканчивается

let m = &mut x;    // ок! разделяемого займа уже нет, можно брать изменяемый
*m += 1;
```

Если бы заём тянулся до `}`, это была бы ошибка по правилу 1. Но `a` после `println!` больше не нужна, её заём «закрылся», и путь для `&mut` свободен. На первых порах ты будешь получать ошибки заимствования — и часто лечатся они просто тем, что лишняя ссылка раньше перестаёт использоваться.

---

## 6. Висячие ссылки запрещены — мостик к временам жизни

Правило 2 («ссылка всегда валидна») означает, что нельзя вернуть ссылку на значение, которое вот-вот умрёт:

```rust
fn dangle() -> &String {            // пытаемся вернуть ссылку...
    let s = String::from("привет");
    &s
}                                    // ...но s уничтожается здесь → ссылка повисла бы
```

```text
error[E0106]: missing lifetime specifier
```

В C++ вернуть указатель/ссылку на локальную переменную можно — компилятор промолчит, а ты получишь висячий указатель и UB. Rust такое не собирает: значение `s` умирает на `}`, ссылка на него стала бы висячей, и это запрещено. Чинится либо возвратом самой `String` (передать владение наружу, как в `ownership.md`), либо аппаратом **времён жизни** — он отвечает на вопрос «как долго эта ссылка обязана оставаться валидной». Это следующий и последний урок ступени: [`lifetimes.md`](./lifetimes.md).

## 7. Срезы — заимствованный «вид» на часть данных

Частный, но важный случай ссылки — **срез** (slice): ссылка на непрерывный участок коллекции, а не на всю её целиком. Ты уже встречал один срез в [`basics.md`](./basics.md): `&str` — это срез строки.

```rust
let s = String::from("привет мир");
let hello: &str = &s[0..12];     // срез байт [0;12) — заимствованный вид в буфер s

let v = vec![10, 20, 30, 40];
let middle: &[i32] = &v[1..3];   // &[i32] — срез вектора: [20, 30]
```

Срез не копирует данные и не владеет ими — это `(указатель, длина)`, смотрящие внутрь чужого буфера. И на него распространяются те же правила заимствования: пока жив срез в коллекцию, менять коллекцию нельзя. `&str` из `basics.md` — ровно это: заимствованный вид в чьи-то строковые байты, оттого он и не владеющий, в отличие от `String`.

## Упражнение

В песочнице (`fn main`), снова с упором на ошибки компилятора:

1. Напиши `fn first_word(s: &String) -> usize`, возвращающую длину до первого пробела (или всю длину). Вызови её и убедись, что строка после вызова **всё ещё доступна** — в этом весь смысл `&`.
2. Напиши `fn grow(v: &mut Vec<i32>)`, добавляющую элемент. Вызови, не забыв `let mut` и `&mut`. Попробуй забыть `mut` у владельца — прочитай ошибку.
3. Воспроизведи инвалидацию: `let first = &v[0]; v.push(99); println!("{first}");`. Поймай ошибку E0502, объясни себе её на пальцах (что было бы в C++).
4. Возьми этот же код и сделай его легальным, **не** убирая `push`: переставь так, чтобы последнее использование `first` было **до** `push` (NLL закроет заём). Прочувствуй, как «последнее использование» двигает границу.

## Шпаргалка

- **`&T`** — разделяемая ссылка (заём на чтение); их можно много; менять через неё нельзя. `&T` — `Copy`.
- **`&mut T`** — изменяемая ссылка (заём на изменение); ровно одна; владелец обязан быть `mut`.
- **Правило:** в каждый момент — ЛИБО много `&T`, ЛИБО одна `&mut T`, не одновременно (**разделение XOR изменение**).
- **Ссылка не владеет** → по выходу из скоупа ничего не уничтожает; `Drop` — у владельца.
- `*r` — разыменование (добраться до значения); для методов/полей дереференс часто авто.
- **NLL:** заём кончается на последнем использовании ссылки, а не на `}`. Многие ошибки лечатся «раньше перестать пользоваться».
- Правила вырезают **гонки данных** и **инвалидацию итераторов** на этапе компиляции.
- Висячая ссылка запрещена (нельзя вернуть ссылку на локальную переменную) → мостик к [`lifetimes.md`](./lifetimes.md).
- **Срез** (`&str`, `&[T]`) — заимствованный `(указатель, длина)`-вид в чужой буфер; подчиняется правилам заимствования.
